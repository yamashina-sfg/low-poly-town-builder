# 実装進捗ログ

拡張ロードマップ（フェーズ7〜）の各フェーズ完了時に記録する。

## フェーズ7：ワールドの拡張（チャンク方式への移行）

- 固定10x10グリッドを、歩いた先に自動生成される「チャンク方式」に変更
  （`src/chunkManager.js`, `src/terrain.js`）。1チャンク=10x10タイル
- キャラのいるチャンクが変わるたび周囲3x3チャンクを生成、離れたチャンクは
  `group.visible=false`で描画スキップ（データはメモリ保持）
- 道路の接続判定・セーブ/ロードもグローバルタイル座標でチャンク横断に対応
  （`src/road.js`, `src/save.js`）。自然生成した木は差分保存不要
- 動作確認：東へ長時間歩いて新チャンクが継ぎ目なく生成されること、
  ページ再読み込み後の読込で家・木が同じ場所に再現されることを確認
  （このフェーズはコミット漏れがあったため、フェーズ8と合わせてここに記録）

## フェーズ8：建築バリエーションの拡大

- 建物3種（お店/井戸/倉庫）、家具4種（ベッド/テーブル/椅子/暖炉）、
  装飾物5種（柵/街灯/ベンチ/花壇/看板）を追加。全て既存のInstancedMesh
  プール方式（`src/primitives.js`に共有プール集約）でプロシージャル生成
- 建築メニューをタブ切り替えUI（建物/家具/装飾/地形）に再構成
  （`src/buildMenu.js`, `index.html`）
- `main.js`に`PROCEDURAL_GENERATORS`レジストリを導入し、種類ごとの
  generate関数とseedソルトを一元管理。デバッグパネルの「建物数」は
  住居/お店/井戸/倉庫の合計を表示するよう拡張
- 動作確認：`npm run dev`でビルドメニューの各タブから全12種類を配置し、
  それぞれ見た目・色が正しく生成されることを確認（Playwrightで自動検証）

## フェーズ9：建物の「内部」システム

- 住居タイルをクリックすると、床・壁・天井のみのシンプルな1部屋の室内
  シーンに切り替わる（`src/interior.js`）。カメラのfar(1000)より遠い
  ワールド座標(3000,0,3000)に部屋を常設し、屋外を隠す処理なしで済ませる設計
- 室内の3x3タイルに家具（ベッド/テーブル/椅子/暖炉）をクリックで配置可能。
  レイアウトは住居タイルごとに`userData.indoorFurniture`として保持され、
  再訪しても同じ配置が復元される
- 「外に出る」ボタンとEscキーで元のワールド・元の位置に戻れる。入退室時は
  カメラを即座にスナップさせ、遠距離テレポートによる空白フレームを回避
- 動作確認：家を建てて中に入り、ベッドとテーブルを配置→外に出る→
  再度入って家具が同じ位置に残っていることをPlaywrightで確認

## フェーズ10：時間経過・生活シミュレーション要素

- ゲーム内時間（現実1秒=ゲーム内1分、24分で1日）を`src/gameTime.js`で管理。
  `dayNightCycle.js`をこのゲーム時計に連動させ、朝夕はオレンジ、昼は白っぽく、
  夜は青暗くなるよう色を変化
- 眠気ステータス（`src/playerStatus.js`）を追加。時間経過で徐々に増加し、
  画面左上に時刻と眠気ゲージを表示
- ベッドに近づく（屋内外どちらでも）と「Eキーで眠る」プロンプトが表示され、
  眠ると時間が朝6時まで進み眠気がリセットされる
- 動作確認：ベッド設置→時間経過で時刻・眠気ゲージが進むことを確認→
  ベッドに近づきEキーで睡眠→時刻が06:00、眠気0%に戻ることをPlaywrightで確認。
  Node.jsで深夜/明け方/正午/夕方の光の色も検証

## フェーズ11：経済・リソースシステム

- 木材・お金の所持リソースを`src/economy.js`で管理。建物・家具・装飾ごとに
  コストを設定し、建築時に不足していれば拒否してステータスメッセージを表示
- 木タイルに近づいてEキーで伐採→木材を入手しタイルは更地に戻る。ベッド／木／
  お店のうち最も近いものに応じてEキーの操作内容と画面下部のプロンプトが変化
- お店に近づいてEキーで簡易売買UIを開き、木材⇄お金の交換ができる
- 画面右上に🪵木材・💰お金を常時表示するリソースパネルを追加
- 動作確認：家を建てて資金が正しく減ることを確認→資金不足で2軒目の建築が
  拒否されることを確認→木を伐採して木材獲得→お店で木材売買が正しく資源を
  増減させることをPlaywrightで確認

## フェーズ12：NPC・生き物

- 汎用の徘徊AI(`src/wander.js`)を作成し、ホーム地点周辺をゆっくり
  ランダムに歩き回るだけのシンプルな挙動を実装
- NPC(`src/npc.js`)はプレイヤーと同じ`createCharacter`ロジックを色違い
  （服・帽子6色パターン）で流用し、町の周囲に6体をランダム配置
- 余力枠として、上空を旋回するだけの簡易な鳥3羽と、同じ徘徊AIを使う
  簡易な犬2匹（`src/creatures.js`）も追加
- 動作確認：起動時にNPC・犬・鳥が周囲に配置され、数秒間隔のスクリーン
  ショット比較でNPCが実際に位置を変えて歩き回っていることを確認

## フェーズ13：ポリッシュ・演出強化

- 足音：歩行中に一定間隔でWeb Audio APIのノイズバーストを鳴らす
  （`ambientAudio.js`に`playFootstep`を追加）。ポップアップアニメーション・
  風音/鳥の声はフェーズ6で実装済み
- パーティクル演出(`src/particles.js`)：木を伐採すると緑の葉っぱが飛び散り、
  水タイルには一定間隔で小さなきらめきが浮かぶ
- ミニマップ(`src/minimap.js`)：追加のWebGLレンダラーを使わず2D Canvasで
  実装。キャラ周囲のタイル種別を色分けした円形ミニマップを画面右下に表示
- 動作確認：木を伐採して葉っぱパーティクルの発生を確認、水タイル設置後に
  きらめきが浮かぶことを確認、ミニマップに地形・水・キャラの向きが正しく
  反映されることをスクリーンショットで確認。コンソールエラーなし
