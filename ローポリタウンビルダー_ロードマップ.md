# ローポリタウンビルダー ロードマップ

## コンセプト

三人称視点でキャラクターが地面を歩き回り、タイルをクリックして
建物・植物・道・水辺などをプロシージャル生成しながら町を作っていく
ローポリ3Dゲーム。参考画像のような低ポリスタイルだが、
素材は画像生成ではなく**すべてThree.jsのジオメトリで手続き的に生成**する。

- カメラ：三人称・キャラ追従（自由回転は画面右クリックドラッグなどで微調整可）
- 移動：WASD / 矢印キーでキャラが地面を歩く
- 建築：地面タイルをクリック → 建築メニュー表示 → 選択した建物が生成される
- スタイル：flat shading、彩度を抑えた配色、InstancedMeshで大量オブジェクトも軽量に

## 技術スタック

- Three.js（生JS、TypeScript不要）
- Vite（開発サーバー・ビルド）
- キャラ・カメラ制御は自作（外部ライブラリのcharacter controllerは使わず、
  シンプルなカプセル型コライダー＋簡易物理でOK）
- 状態管理はシンプルなJSオブジェクト（グリッド配列 + タイル情報）
- セーブはlocalStorage（JSON化したグリッドデータ）

## 全体アーキテクチャ

```
World
 ├─ Terrain（N x N グリッド、各セルにタイル種を保持）
 ├─ Character（三人称操作キャラ、地面を歩く）
 ├─ Camera（キャラに追従、少し後ろ上空から見下ろす角度）
 ├─ BuildSystem（レイキャストでクリックしたタイル座標を取得 → 建築メニュー）
 ├─ Generators
 │   ├─ generateBuilding(seed, type) → THREE.Group
 │   ├─ generateTree(seed, type) → THREE.Group
 │   ├─ generateWater(width, height) → THREE.Mesh（半透明・波アニメ）
 │   └─ generateProp(type) → 街灯・ベンチ・標識など
 └─ UI（建築メニュー、所持リスト、デバッグパネル）
```

## フェーズ0：ロジック検証（Node.jsのみ、描画なし）

3D実装の前に、グリッドとタイル管理のロジックだけを
Node.jsスクリプトで検証する。

**Claude Codeプロンプト：**
```
Node.jsスクリプトで、N x Nのグリッドを表すタウンデータ構造を実装してください。

- 各セルは { x, y, tileType, building: null } を持つ
- tileTypeは 'empty' | 'grass' | 'road' | 'water' のいずれか
- 初期状態はすべて'grass'で、外周に'water'を配置するテスト関数
- 特定の座標に'building'をセットする関数 placeBuilding(x, y, buildingData)
- 隣接する道路がないと建物を置けない、などのバリデーション関数
  （最初はシンプルでOK、後で拡張）
- console.logでグリッドを文字グリッドとして可視化する関数

これらをテストするサンプル実行コードも書いて、正しく動くか確認してください。
```

## フェーズ1：最小プロトタイプ（キャラが歩ける地面を作る）

**Claude Codeプロンプト：**
```
Three.js + Viteでプロジェクトをセットアップしてください。

- 平面の地面（20x20程度、flat shadingの緑色）
- 簡易キャラクター（カプセル+球のローポリな人型でOK、詳細は後回し）
- WASDキーでキャラが地面上を移動できる
- カメラはキャラの斜め後ろ上空から追従する三人称視点
  （キャラの移動方向に応じてカメラも滑らかに追従）
- HemisphereLight + DirectionalLightで低ポリらしい陰影
- 空は水色の背景色

まずキャラが地面を自由に歩き回れる状態まで作ってください。
```

## フェーズ2：タイルグリッドと建築メニュー

**Claude Codeプロンプト：**
```
フェーズ1の地面を、10x10のタイルグリッドに分割してください。

- 各タイルは1つのPlaneGeometryセグメントとして識別できるようにする
  （マウスクリック時にレイキャストでどのタイルかを特定）
- タイルにマウスホバーすると、そのタイルがハイライトされる
  （色を明るくする、または枠線を表示）
- タイルをクリックすると、画面に建築メニュー（HTML UIでOK）が表示される
  - メニュー項目：家、木、道路、水、更地に戻す
- メニューから選択すると、そのタイルの中心に対応するオブジェクトが
  仮置き（今はBoxGeometryのダミーでOK）される
- 三人称キャラの移動を妨げないよう、UIはオーバーレイで表示する

これでクリック→建築の基本フローを完成させてください。
```

## フェーズ3：プロシージャル建物・木ジェネレーター

**Claude Codeプロンプト：**
```
フェーズ2のダミーオブジェクトを、実際のプロシージャル生成に置き換えます。

generateBuilding(seed, type)：
- 立方体を1〜3段積み重ねて高さのバリエーションを出す
- 各段の色は彩度を抑えた低ポリらしい配色パレットからランダム選択
- 屋根は三角屋根（ConeGeometry）か平屋根をランダムで選択
- flat shading必須
- 同じseedなら毎回同じ建物になるようにする（決定論的乱数）

generateTree(seed, type)：
- 針葉樹タイプ（円柱の幹＋円錐を2〜3段）
- 広葉樹タイプ（円柱の幹＋球を3〜4個ランダムオフセットで重ねる）
- 緑〜オレンジ〜黄色のバリエーションをランダムに混ぜる

建築メニューから「家」を選んだらgenerateBuildingが、
「木」を選んだらgenerateTreeが呼ばれて、該当タイルに配置されるようにしてください。
```

## フェーズ4：水・道路・パフォーマンス最適化

**Claude Codeプロンプト：**
```
以下を実装してください。

1. 水タイル：半透明の青いPlaneGeometry、頂点シェーダーか
   簡易なsin波アニメーションで揺れているように見せる
2. 道路タイル：グレーの平面＋白い破線のテクスチャorジオメトリ、
   隣接する道路タイルと自動的に接続されて見えるようにする
   （角・直線・T字・十字を判定して形状を変える簡易ロジックでOK）
3. パフォーマンス対策：
   同じ種類の建物・木が増えてきたらInstancedMeshに統合し、
   FPSが60を維持できるか確認する
   （画面左上にFPSカウンターを表示）
```

## フェーズ5：セーブ・ロード・デバッグUI

**Claude Codeプロンプト：**
```
以下のUI・機能を追加してください。

- 画面左上に参考画像のようなデバッグパネル：
  タイル数、建物数、木の数、FPS、インスタンス数を表示
- 「保存」ボタンでグリッド状態をlocalStorageにJSON保存
- 「読込」ボタンで保存データから町を復元
- 「リセット」ボタンで更地に戻す
- シード値入力欄：入力したシードで町の見た目（建物の配色パターンなど）
  を一括で変えられるようにする
```

## フェーズ6：キャラ・演出のブラッシュアップ（余力があれば）

- キャラクターの見た目をローポリ人型としてちゃんと作り込む（帽子・服の色を変更可能に）
- 建物設置時のポップアップアニメーション（下から生えてくる演出）
- BGM・環境音（鳥の声、風の音）
- 昼夜サイクル（DirectionalLightの角度と色を時間で変化）

## 素材生成の方針まとめ

| 要素 | 生成方法 |
|---|---|
| キャラクター | プロシージャル（カプセル・球・円柱の組み合わせ） |
| 建物 | プロシージャル（Boxの積み重ね＋ランダム配色） |
| 木・植物 | プロシージャル（円柱＋円錐/球） |
| 水・海 | プロシージャル（半透明平面＋波アニメ） |
| 道路 | プロシージャル（平面＋接続ロジック） |
| UI装飾アイコン等 | 必要ならGeminiで簡易生成してもOK（ゲーム本体の3Dアセットには使わない） |

## 進め方

Claude Codeに対して「このロードマップのフェーズ0から順番に実装して、
各フェーズが終わったら一度止まって報告して」と依頼する。
節目ごとに動作確認し、次のフェーズに進めるか判断する。
