# ローポリタウンビルダー 拡張ロードマップ（フェーズ7〜）

## 現状（フェーズ0〜6完了時点）

- 10x10固定グリッド、三人称キャラ操作
- タイルクリックで建築メニュー（家/木/道路/水/更地に戻す）
- プロシージャル生成（建物・木にランダムバリエーション）
- 水の波アニメーション、道路の接続ロジック、InstancedMesh最適化
- セーブ/ロード（localStorage）、シード値、キャラの服・帽子の色変更、BGM

## 今回の課題認識

1. ワールドが狭い（10x10で固定、探索の広がりがない）
2. 生活感がない（寝る、家具、時間経過などの「暮らし」の要素が皆無）
3. 建築バリエーションが少ない（家しか建てられない）

これらを解決するため、フェーズ7以降を「広さ」「深さ」「バリエーション」の
3方向に分けて拡張する。

---

## フェーズ7：ワールドの拡張（チャンク方式への移行）

現状の固定10x10グリッドを、歩いた先に自動で新しいエリアが
生成される「チャンク方式」に作り替える。

**Claude Codeプロンプト：**
```
現在10x10の固定グリッドになっているワールドを、
チャンク方式（歩いた先に自動で新しいグリッドが生成される仕組み）に
作り替えてください。

- 1チャンク = 10x10タイル（現状のサイズを維持）
- キャラがチャンクの端に近づいたら、隣接するチャンクを自動生成
- 各チャンクはシード値 + チャンク座標から決定論的に生成する
  （同じ場所に戻ったら同じ内容が再現される）
- 生成済みチャンクのタイル・建物データはメモリ上に保持し続ける
  （離れても消えない、離れすぎたチャンクは描画をスキップしてOK）
- チャンクの境界がわからないよう、地面のつなぎ目を自然にする
- 保存/読込機能も複数チャンクに対応させる
```

## フェーズ8：建築バリエーションの拡大

「家」の1種類しかない建築メニューを、複数の建物種類・
家具・装飾物に拡張する。

**Claude Codeプロンプト：**
```
建築メニューを拡張してください。

新しい建築カテゴリ：
- 建物：住居（今の家）、お店、井戸、倉庫
- 家具（建物の中に置くもの）：ベッド、テーブル、椅子、暖炉
- 装飾物：柵、街灯、ベンチ、花壇、看板

各建物・家具・装飾物ごとにgenerate関数を作成し、
プロシージャルにバリエーションを出してください
（フェーズ3のgenerateBuilding/generateTreeと同じ設計思想で）。

建築メニューはカテゴリ分けして表示し、
選びやすいUIにしてください（タブ切り替えなど）。
```

## フェーズ9：建物の「内部」システム

建物をクリックすると中に入れるようにし、
家具を配置できる室内空間を作る。

**Claude Codeプロンプト：**
```
建物（住居）に「内部」の概念を追加してください。

- 建物をクリックすると、外観の代わりに簡易的な室内シーンに切り替わる
  （最初はシンプルな1部屋の箱でOK、床・壁・天井のみ）
- 室内にフェーズ8で作った家具（ベッド、テーブル、椅子など）を配置できる
- 「外に出る」ボタンかキー操作で元のワールドに戻れる
- キャラも室内用の縮小スケールで表示され、室内を歩き回れる
```

---

## フェーズ10：時間経過・生活シミュレーション要素

昼夜サイクルとキャラのステータス（眠気など）を追加し、
「寝る」という行動に意味を持たせる。

**Claude Codeプロンプト：**
```
時間経過システムを実装してください。

- ゲーム内時間（例：現実の1分 = ゲーム内1時間、24時間サイクル）
- 昼夜に応じてDirectionalLightの角度と色を変化させる
  （朝はオレンジ、昼は白、夜は青暗く）
- キャラに「眠気」ステータスを追加し、時間経過で徐々に増加
- ベッド（フェーズ8で作成）に近づいて「眠る」操作をすると、
  時間が朝まで進み、眠気がリセットされる
- 画面左上に現在時刻と眠気ゲージを表示
```

## フェーズ11：経済・リソースシステム

素材収集とお金の概念を追加し、建築に「コスト」を持たせることで
プレイに目的意識を持たせる。

**Claude Codeプロンプト：**
```
簡易的な経済システムを実装してください。

- 木を伐採する（木タイルに近づいてアクションキー）と「木材」を入手
- 建物を建てるのに木材・お金などのコストを要求するようにする
- お店（フェーズ8の建物）で素材を売買できる簡易UI
- 所持リソース・お金を画面に表示
```

---

## フェーズ12：NPC・生き物

町に住人や動物を配置し、賑わいを出す。

**Claude Codeプロンプト：**
```
簡易的なNPC（住人）システムを実装してください。

- プロシージャルに生成されたキャラクター（プレイヤーと同じ生成ロジック
  を流用し、色違いにする）を数体、町にランダム配置
- NPCは決まった範囲をゆっくり徘徊するだけのシンプルな挙動でOK
- 余力があれば、鳥や犬などの小動物も追加
```

## 進め方の推奨

1. まずフェーズ7（ワールド拡張）を最優先で実施する
   → 「狭い」という一番の不満をまず解消する
2. 次にフェーズ8（建築バリエーション）
   → 「家しか建てられない」を解消する
3. フェーズ9・10（内部システム・生活シミュレーション）で
   「寝る」を実現する
4. フェーズ11以降は余力を見ながら順次

各フェーズごとに動作確認→GitHub push→Vercel自動反映、を継続する。
